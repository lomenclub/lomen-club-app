# Contributing

## Purpose
Provide guidance for contributing changes to the Lomen Club repository.

## Audience
All contributors.

## Prerequisites
- Access to the repository.
- Familiarity with the system overview.

## Contribution Workflow
1. Create a feature branch.
2. Make changes with clear commit messages.
3. Run lint/tests relevant to your changes.
4. Update documentation if behavior changes.
5. Open a pull request with a summary and testing notes.

## Documentation Expectations
- Use the docs structure under `docs/`.
- Every doc should include: purpose, audience, prerequisites, and last verified note.
- Prefer linking to canonical docs rather than duplicating content.

## Optional Docs Linting
You can run Markdown linting with:
```bash
npx markdownlint-cli2 "**/*.md"
```
The config lives in `.markdownlint.jsonc`.
